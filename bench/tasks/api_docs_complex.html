Vaultline Platform API v3

Base URL: https://api.vaultline.example.com/v3

The Vaultline API provides programmatic access to secret management, vault operations, and audit capabilities. All API endpoints require authentication and return JSON responses.

Authentication

All API requests must include an API key in the Authorization header using the Bearer scheme:

curl -H "Authorization: Bearer vlt_sk_live_abc123def456" \
     https://api.vaultline.example.com/v3/vaults

API keys are scoped to specific permissions. Use the dashboard to generate keys with the minimum required permissions for your use case.

API Key Types
Key PrefixScopeDescription
vlt_sk_live_ProductionFull access to live vaults and secrets
vlt_sk_test_SandboxAccess to test vaults only, safe for development
vlt_pk_PublicRead-only access to public vault metadata

API requests are subject to rate limiting based on your plan tier:

Rate limit headers are included in every response:

X-RateLimit-Limit: 600

Error Handling

The API uses conventional HTTP status codes. Error responses include a JSON body with structured error information:

{
  "error": {
    "code": "vault_not_found",
    "message": "No vault exists with ID 'vlt_abc123'",
    "status": 404,
    "request_id": "req_7f3a9b2c"
  }
}
Common Error Codes
StatusCodeDescription
400invalid_requestRequest body or parameters are malformed
401authentication_failedMissing or invalid API key
403insufficient_permissionsAPI key lacks required scope
404not_foundResource does not exist
429rate_limitedToo many requests, retry after cooldown
500internal_errorServer error, contact support with request_id

Create Vault

POST
/v3/vaults

Creates a new vault for storing secrets. Each vault is an isolated namespace with its own access controls and encryption keys.

Request Body

ParameterTypeRequiredDescription
namestringYesHuman-readable name (3–64 chars, alphanumeric and hyphens)
descriptionstringNoDescription of the vault's purpose (max 256 chars)
environmentstringNoOne of: production, staging, development. Default: production
encryptionstringNoEncryption algorithm: aes-256-gcm (default) or chacha20-poly1305
tagsobjectNoKey-value metadata tags (max 10 tags)

Example Request

curl -X POST https://api.vaultline.example.com/v3/vaults \
  -H "Authorization: Bearer vlt_sk_live_abc123" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "production-secrets",
    "description": "Production environment secrets",
    "environment": "production",
    "tags": {"team": "platform", "service": "auth"}
  }'

Response (201 Created)

{
  "id": "vlt_a1b2c3d4",
  "name": "production-secrets",
  "description": "Production environment secrets",
  "environment": "production",
  "encryption": "aes-256-gcm",
  "secret_count": 0,
  "tags": {"team": "platform", "service": "auth"},
  "created_at": "2025-06-15T10:30:00Z",
  "updated_at": "2025-06-15T10:30:00Z"
}

List Vaults

GET
/v3/vaults

Returns a paginated list of all vaults accessible to the authenticated API key.

Query Parameters

ParameterTypeDefaultDescription
pageinteger1Page number
per_pageinteger20Results per page (max 100)
environmentstringallFilter by environment
sortstringcreated_atSort field: name, created_at, updated_at

Example Request

curl "https://api.vaultline.example.com/v3/vaults?environment=production&per_page=10" \
  -H "Authorization: Bearer vlt_sk_live_abc123"

Response (200 OK)

{
  "data": [
    {"id": "vlt_a1b2c3d4", "name": "production-secrets", "secret_count": 42},
    {"id": "vlt_e5f6g7h8", "name": "api-keys", "secret_count": 18}
  ],
  "pagination": {
    "page": 1,
    "per_page": 10,
    "total": 2,
    "total_pages": 1
  }
}

Store Secret

POST
/v3/vaults/{vault_id}/secrets

Stores a new secret in the specified vault. The secret value is encrypted at rest using the vault's encryption algorithm. Maximum secret size is 64KB.

Request Body

ParameterTypeRequiredDescription
keystringYesSecret key name (unique within vault, 1–128 chars)
valuestringYesThe secret value (max 64KB)
content_typestringNoMIME type hint: text/plain, application/json, application/x-pem-file
expires_atstringNoISO 8601 expiration timestamp
rotation_policyobjectNoAutomatic rotation settings (see Rotate Secret)

Example Request

curl -X POST "https://api.vaultline.example.com/v3/vaults/vlt_a1b2c3d4/secrets" \
  -H "Authorization: Bearer vlt_sk_live_abc123" \
  -H "Content-Type: application/json" \
  -d '{
    "key": "DATABASE_URL",
    "value": "postgresql://user:pass@db.example.com:5432/mydb",
    "content_type": "text/plain",
    "expires_at": "2026-01-01T00:00:00Z"
  }'

Response (201 Created)

{
  "id": "sec_x1y2z3",
  "key": "DATABASE_URL",
  "vault_id": "vlt_a1b2c3d4",
  "version": 1,
  "content_type": "text/plain",
  "created_at": "2025-06-15T10:45:00Z",
  "expires_at": "2026-01-01T00:00:00Z"
}

Get Audit Log

GET
/v3/audit

Returns a time-ordered log of all API actions performed within your organization. Audit logs are retained for 90 days on Pro plans and 365 days on Enterprise plans.

Example Request

curl "https://api.vaultline.example.com/v3/audit?since=2025-06-01T00:00:00Z&limit=50" \
  -H "Authorization: Bearer vlt_sk_live_abc123"

Response (200 OK)

{
  "events": [
    {
      "id": "evt_001",
      "action": "secret.read",
      "vault_id": "vlt_a1b2c3d4",
      "secret_key": "DATABASE_URL",
      "actor": "api_key:vlt_sk_live_abc123",
      "ip_address": "203.0.113.42",
      "timestamp": "2025-06-15T10:50:00Z"
    }
  ],
  "has_more": true,
  "cursor": "cur_abc123"
}
