Configuration Guide

Overview

Vortex uses a layered configuration system that supports multiple sources. Configuration is resolved in the following priority order (highest first):

  1. Programmatic overrides passed at runtime
  2. Environment variables prefixed with VORTEX_
  3. Configuration file (vortex.toml or vortex.yaml)
  4. Built-in defaults

All configuration values are validated at startup. Invalid values cause a clear error message with the expected type and constraints.

Configuration File

The recommended way to configure Vortex is through a vortex.toml file in your project root. Here is a complete example:

[vortex]
name = "my-project"
version = "1.0.0"
debug = false

[vortex.server]
host = "0.0.0.0"
port = 8080
workers = 4
graceful_shutdown_timeout = 30

[vortex.database]
url = "postgresql://localhost:5432/mydb"
pool_size = 20
pool_timeout = 30
echo_sql = false

[vortex.cache]
backend = "redis"
url = "redis://localhost:6379/0"
ttl = 3600
prefix = "vortex:"

[vortex.logging]
level = "INFO"
format = "json"
file = "logs/vortex.log"

Note: YAML configuration files are also supported. If both vortex.toml and vortex.yaml exist, the TOML file takes precedence.

Environment Variables

Any configuration value can be set via environment variables using the VORTEX_ prefix and double underscores for nesting:

# Set server port
export VORTEX_SERVER__PORT=9090

# Set database URL
export VORTEX_DATABASE__URL="postgresql://prod:5432/mydb"

# Enable debug mode
export VORTEX_DEBUG=true

Environment variables are especially useful for secrets and environment-specific values that should not be committed to version control.

Programmatic Configuration

For maximum flexibility, you can configure Vortex programmatically in your application code:

from vortex import Vortex, Config

config = Config(
    server=Config.Server(host="0.0.0.0", port=8080, workers=4),
    database=Config.Database(url="postgresql://localhost/mydb", pool_size=20),
    cache=Config.Cache(backend="redis", url="redis://localhost:6379/0"),
)

app = Vortex(config=config)

# Override specific values
app.config.server.port = 9090
app.config.logging.level = "DEBUG"

Warning: Programmatic configuration changes after app.start() will not take effect for already-initialized components. Restart the application to apply changes.

Validation

Vortex validates all configuration at startup using typed schemas. If a configuration value is invalid, you will see a clear error:

ConfigurationError: Invalid value for 'server.port'
  Expected: integer between 1 and 65535
  Got: -1
  Source: environment variable VORTEX_SERVER__PORT

You can also validate configuration programmatically before starting:

from vortex import Config

config = Config.from_file("vortex.toml")
errors = config.validate()

if errors:
    for error in errors:
        print(f"Config error: {error}")
else:
    print("Configuration is valid")
