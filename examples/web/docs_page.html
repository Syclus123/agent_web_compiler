API Reference

Welcome to the CloudPlatform API. This reference documents all available endpoints, authentication methods, and error codes.

Authentication

All API requests require authentication using an API key. Include your key in the Authorization header:

curl -H "Authorization: Bearer YOUR_API_KEY" https://api.cloudplatform.io/v1/resources

API keys can be generated from your dashboard.

Endpoints

List Resources

Returns a paginated list of all resources in your account.

GET /v1/resources?page=1&limit=20

Parameters

ParameterTypeRequiredDescription
pageintegerNoPage number (default: 1)
limitintegerNoItems per page (default: 20, max: 100)
statusstringNoFilter by status: active, archived, deleted
sortstringNoSort field: created_at, name, status

Response

{
  "data": [
    {
      "id": "res_abc123",
      "name": "Production DB",
      "status": "active",
      "region": "us-east-1",
      "created_at": "2026-01-15T10:30:00Z"
    }
  ],
  "pagination": {
    "page": 1,
    "limit": 20,
    "total": 156,
    "has_more": true
  }
}

Create Resource

Creates a new resource in your account.

POST /v1/resources

Request Body

{
  "name": "My Resource",
  "type": "compute",
  "region": "us-east-1",
  "config": {
    "cpu": 4,
    "memory_gb": 16,
    "storage_gb": 100
  }
}

Delete Resource

DELETE /v1/resources/{id}

Permanently deletes a resource. This action cannot be undone.

Error Handling

The API uses standard HTTP status codes. Error responses include a JSON body with details:

Status CodeMeaningDescription
400Bad RequestInvalid request parameters
401UnauthorizedMissing or invalid API key
403ForbiddenInsufficient permissions
404Not FoundResource does not exist
429Too Many RequestsRate limit exceeded
500Internal ErrorServer-side error

Rate Limits

API requests are limited to 1,000 requests per minute per API key. Rate limit headers are included in every response:

  • X-RateLimit-Limit: Maximum requests per minute
  • X-RateLimit-Remaining: Remaining requests in current window
  • X-RateLimit-Reset: Unix timestamp when the limit resets

Tip: Use exponential backoff when you receive a 429 response. Start with a 1-second delay and double on each retry.
